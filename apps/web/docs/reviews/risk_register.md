# 리스크 대장 (Risk Register)

본 프로젝트의 운영 안정성, 보안, 데이터 정합성에 중대한 악영향을 미칠 수 있는 식별 리스크 장부입니다.

| 리스크 ID | 위험명 | 분류 | 발생 가능성 | 영향력 | 조치 방안 (Mitigation) |
| :--- | :--- | :--- | :---: | :---: | :--- |
| **RISK-001** | 운영 연동 키 유출 | 보안 | **High** | **Critical** | 평문 시크릿 즉시 무효화 후 재발행. 설정 코드를 프로퍼티 `${}`를 활용한 환경변수 동적 삽입 방식으로 전면 개정. |
| **RISK-002** | 세션 탈취를 통한 토큰 유도 | 보안 | Medium | High | `HttpSession`으로 JWT Access/Refresh 토큰을 보존하고 인출 검사하는 메커니즘 차단. |
| **RISK-003** | 회원 탈퇴 정보 노출 | 개인정보 | Low | Medium | 논리 삭제 엔티티 자동 격리를 위한 `@SQLRestriction` 설정 적용. |
| **RISK-004** | 전북대 학사 XML 파싱 실패 | 운영 | Medium | Medium | 학사 페이지 템플릿 변경 시 즉각 전파 감지 및 슬랙/디스코드 에러 채널 알림을 위한 폴백 크롤러 작성. |
| **RISK-005** | 배치 메모리 누수로 인한 호스트 붕괴 | 안정성 | Low | High | Docker Compose 파일에 CPU 사용량 한계 및 메모리 최대 허용 Limits 기재. |
